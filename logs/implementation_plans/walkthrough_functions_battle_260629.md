# 战斗环境与团队增益实现报告

## 修改内容概览

1. **敌人实体系统重构** (`backend/function/battle/enemy.py`)
   - 为 `Enemy` 基类注入了全局状态字段：`is_cced`（是否被控）、`global_fragile`（全局脆弱）、`global_arts_flat_dmg`（全局法伤附加）。
   - 添加了快速生成从 0 防 0 抗到 5000 防 100 抗的 6 种标准木桩模型（以及专为本次协同测试生成的 `99号：2000防40抗木桩`）。
2. **抽象基类标准化** (`backend/function/logic/base_operator.py`)
   - 在基类 `Operator` 补充了虚方法 `get_team_buffs()`，供所有提供团队收益的“拐”调用。
3. **战斗引擎的 Monkey Patching (核心亮点)** (`backend/function/battle/engine.py`)
   - `BattleEnvironment.simulate()` 在战斗启动前，会抓取在场所有干员通过 `get_team_buffs` 暴露的团队增益，包括最大脆弱值、总减抗、受控状态等。
   - **动态劫持底层公式**：拦截了 `formulas.py` 的物理和法伤公式，自动在每次伤害结算时附加全局法伤与最终脆弱乘区。这使得我们不必去碰此前大模型生成的 129 个干员的独立算法，实现了 100% 的底层降维覆盖。
4. **三人小队的接口改造**
   - **逻辑斯 (RE03)**：暴露了全场法伤附伤和减抗。
   - **铃兰 (R172)**：三技能暴露了 40% 的脆弱以及全覆盖的停顿（`is_cced=True`）。
   - **仇白 (YD20)**：天赋 1 的触发条件（是否受控）由原本闭塞的写死判定，改为了对 `enemy.is_cced` 环境状态的读取。

## 测试结果验证

我们执行了 `backend/main/run_team_synergy.py`，让仇白、逻各斯、铃兰开启三技能共同攻击 `2000防御/40法抗` 的木桩。结果如下：

### 单打独斗总伤（没有团队环境增益）
- **仇白 (S3)**: 总伤 39,004.1
- **逻各斯 (S3)**: 总伤 47,619.6
- **铃兰 (S3)**: 总伤 0.0（不攻击）
> **单打团队总和**: 86,623.7

### 协同环境总伤（Monkey Patching 注入环境后）
- **自动提取的全队面板**:
  - `fragile`: 0.4
  - `res_shred`: 10.0
  - `arts_flat_dmg`: 150.0
  - `is_cced`: True
- **仇白 (S3)**: 总伤暴涨至 **70,610.9**（因 100% 触发第一天赋附加法伤，再吃满脆弱、减抗和附加平A伤害）
- **逻各斯 (S3)**: 总伤提升至 **53,859.9**（吃满脆弱带来的独立乘区）
- **铃兰 (S3)**: 0.0
> **协同团队总和**: **124,470.8**

**结论**: 协同增伤幅度达到了惊人的 **+43.7%**！
