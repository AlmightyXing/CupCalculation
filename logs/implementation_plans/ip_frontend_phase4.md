# 🎯 前端开发执行计划：阶段 4（干员详情能力展板）

根据《前端设计与开发任务单》，当前进入阶段 4。我将指派 **“UI 与数据 Agent”** 接管此阶段。本阶段的核心目标是构建单体干员的全方位数据透视页面，将杯级计算的底层细节通过可视化的方式呈现给用户。

## 1. 整体架构与开发目标

本阶段聚焦于 `frontend/pages/detail/detail` 页面的深度开发。

### 阶段 4 核心任务清单

1. **干员头部信息区 (Hero Section)**：

   - 顶部保留自定义导航栏（带返回键）。
   - 主视觉展示干员高清半身/全身立绘（使用网络 CDN 链接或本地占位图）。
   - 悬浮叠加展示干员名称、职业图标、以及最醒目的“总杯级评定徽章”（如：超大杯）。
2. **雷达图与能力值看板 (Radar & Ability Board)**：
   - **雷达图 (`components/radar-chart`)**：复用并扩展已有的 ECharts (`ec-canvas`)，绘制展示该干员在“物理挂机”、“法术挂机”、“物理决战”、“法术决战”、“生存抗伤”等 5-6 个维度上的综合评分雷达图。
   - **细分成绩单**：列表式展示干员在不同护甲/法抗环境下的具体排名与总伤（例如：1000甲 决战排名 #1，总伤 125,430）。
3. **下钻明细与透明公式弹窗 (Popup/Modal)**：
   - 实现一个可复用的微信小程序底部弹出层组件 (`components/bottom-sheet` 或直接手写 Modal)。
   - 点击成绩单上的任意一条数据，触发该弹窗。
   - 弹窗内详细展示：**当前敌人环境基准**、**该干员本次计算所吃到的团队 Buff（如脆弱/法伤附加等，呼应后端架构）**、以及**完整的计算公式（理论最大值推演）**。
