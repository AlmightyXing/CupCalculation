# 🎯 前端开发执行计划：阶段 4 优化重构（干员详情页）

根据您最新的《前端Phase4详细设计方案.md》，我们已彻底放弃了初期的雷达图与弹窗设计，转而采用**强数据驱动的、高密度的表格化呈现**方式，完美契合硬核计算器的定位。

## 1. 整体架构与开发目标

本阶段聚焦于 `frontend/pages/detail/detail` 页面的构建，页面从上至下分为三大核心模块：

### 模块一：干员主视觉区 (Hero Section)

- **立绘切换机制**：在左上角实现“初始立绘 / 精二立绘”的 Toggle 按钮。
- **排版要素**：包含 6 星稀有度标识、中/英文名、主/子职业（带占位图标，提取 `character_type`）、部署位与标签等信息。
- **背景层**：采用沉浸式布局，导航栏（带返回键及“干员名称-详细信息”）固定在顶部不随页面滚动。

### 模块二：干员能力排名摘要 (Rank Summary)

- 简明扼要地展示该干员的核心三榜排名（总榜排名、挂机排名、决战排名）。

### 模块三：硬核属性表格矩阵 (Data Tables)

我们将严格参照您规定的 Markdown 表格样式以及 `#C1C6D6` 的配色比例（首行首列 60% 透明度/其他 20% 透明度），实现四张原生重构的 Flex 表格：

1. **特性表 (Trait)**：实现横向单元格合并以使长描述居中，同时隐藏特定单元格的分隔线。
2. **属性表 (Stats)**：渲染生命、攻击、防御、法抗在“精二满级”与“信赖加成”下的双列对比。
3. **天赋表 (Talents)**：渲染第一与第二天赋的名称及详细描述。
4. **技能表 (Skills)**：每个技能都有独立的区块，包含“技能基础数值表格”（包含初始/消耗/持续等并按要求合并居中 `skill_type`）以及“伤害产出表格”（包含 DPS 与总伤的具体排名）。

## ⚠️ User Review Required

> [!IMPORTANT]
> **关于复杂表格的合并格 (Colspan/Rowspan) 实现**
> 微信小程序无法直接使用 HTML 的 `<table colspan="2">` 语法。为了完美实现您设计案中类似“描述及描述内容与其左侧空列合并为一个单元”这种跨列/跨行居中需求，我们将高度依赖 Flex 布局和精细的 CSS Border 控制来模拟表格效果。
> **Mock 数据结构**
> 由于您的设计案对数据要求极度详实（包括英文名、立绘 URL、技能初始/消耗SP等，这些在目前的 `YD30_赤刃明霄陈.json` 中部分字段缺失，如英文名、初始SP等），本阶段在前端 `detail.js` 中，我将为您手捏一份**极其完整的、包含所有需要字段的 Mock 数据模型**，用以验证表格渲染。
