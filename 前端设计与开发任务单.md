# 前端设计与开发任务单

根据《杯级计算器》PRD v1.3 及初期原型图构建的前端（微信小程序原生框架）开发流。

## 1. 工程初始化与全局设置 (✓ 已完成)

- [x] 微信开发者工具初始化项目，配置 `app.json`（定义全局路由与窗口样式）。
- [x] 设定全局 UI 设计规范（Color Palette、排版规则、组件主题），建立通用的 `app.wxss`。
- [x] 封装网络请求模块（统一处理后端 API `/api/simulate` 的数据拉取与错误拦截，`utils/request.js`）。
- [x] 准备公用组件：顶栏/底栏导航、统一的加载与空状态反馈组件。

## 2. 首页：排行榜单流 (Main Interface)

- [x] **全干员综合杯级榜单**：
  - [x] 顶部分类标签切换（总杯、挂机、决战等）。
  - [x] 列表单行卡片组件：展示杯级标签（超大杯/大杯等）、排名、头像、名称、职业图标、关键数值（DPS/总伤）。
  - [ ] 版本更迭标识（NEW 或 排名升降标识）。
- [ ] **数据可视化切换**：
  - [ ] 在单项子榜单中加入列表/柱状图切换按钮。
  - [ ] 集成可视化图表库（如 ECharts for WeiXin），绘制前 10 名干员数据的横向/纵向条形图。

## 3. 搜索与发现模块 (Search)

- [x] **独立搜索页**：
  - [x] 搜索输入框与历史搜索记录。
  - [x] 联想与模糊查询逻辑（目前前端内置词库，支持搜“水陈”、“42”）。
- [x] **搜索结果流**：复用首页的干员卡片组件以呈现搜索结果。

## 4. 干员详情能力展板 (Operator Detail)

- [x] **干员头部信息区**：高清半身/全身立绘，干员名称、职业、标签展现。
- [x] **能力值看板与面板详情**：
  - [x] 绘制精美的暗黑风属性面板。
  - [x] 以表格形式列出并合并干员特性、天赋及技能。
- [ ] **下钻明细与透明公式弹窗**：
  - [ ] 支持点击各项数据项，弹出具体运算公式或环境基准提示（例如：`1000甲 * 2.6最终倍率`），展示数据的由来。

## 5. 交互与联调

- [x] 对接后端提供的各个 HTTP API 接口（已对接单个干员详情 `/api/operator/{name}`）。
- [ ] 数据流与页面响应的真机测试（处理大数据量渲染卡顿与瀑布流懒加载优化）。
- [ ] 对接云开发的静态资源存储（云存干员头像、立绘等图片，缩减本地包体积）。

## 🚀 后续开发规划：Phase 6 ~ Phase 7

### Phase 6: 核心计算引擎接入与全局排位演算 (Core Calculation Engine)
- [ ] **完善战斗模拟逻辑**：在 `backend/function/battle/` 模块下，跑通针对不同技能类型的 `simulator`，兼容瞬发、弹药、挂机等技能。
- [ ] **后端接口联调**：补充调用 `/api/simulate` 或增设批量跑分接口，让首页的排行榜拥有**真实排名**，而非固定 Mock 数据。
- [ ] **前后台联动**：将运算出的动态数据（DPS、总伤、技能名次）灌入前端详情页的“伤害表格”与首页榜单中，替代当前的 `演算中` 占位符。

### Phase 7: 细节打磨、可视化与云端发布 (Polishing & Deployment)
- [ ] **云存储资源替换**：将现在的静态素材（背景图、假立绘）替换为 COS/OSS 的云端链接，完善真实的头像获取机制。
- [ ] **图表化分析**：在榜单引入 ECharts 柱状图对比，直观展示 DPS 差距断层。
- [ ] **交互补充**：实现公式详解弹窗，加入长列表无限滚动加载（Virtual List），解决 300+ 数量干员的渲染性能。
- [ ] **异常与体验**：真机环境下网络请求的骨架屏（Skeleton）、接口降级处理。
