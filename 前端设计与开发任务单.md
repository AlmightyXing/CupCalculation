# 前端设计与开发任务单

根据《杯级计算器》PRD v1.3 及初期原型图构建的前端（微信小程序原生框架）开发流。

## 1. 工程初始化与全局设置

- [ ] 微信开发者工具初始化项目，配置 `app.json`（定义全局路由与窗口样式）。
- [ ] 设定全局 UI 设计规范（Color Palette、排版规则、组件主题），建立通用的 `app.wxss`。
- [ ] 封装网络请求模块（统一处理后端 API `/api/simulate` 的数据拉取与错误拦截）。
- [ ] 准备公用组件：顶栏/底栏导航、统一的加载与空状态反馈组件。

## 2. 首页：排行榜单流 (Main Interface)

- [ ] **全干员综合杯级榜单**：
  - [ ] 顶部分类标签切换（挂机、决战、生存等）。
  - [ ] 列表单行卡片组件：展示杯级标签（超大杯/大杯等）、排名、头像、名称、职业图标、关键数值（DPS/总伤）。
  - [ ] 版本更迭标识（NEW 或 排名升降标识）。
- [ ] **数据可视化切换**：
  - [ ] 在单项子榜单中加入列表/柱状图切换按钮。
  - [ ] 集成可视化图表库（如 ECharts for WeiXin），绘制前 10 名干员数据的横向/纵向条形图。

## 3. 搜索与发现模块 (Search)

- [ ] **独立搜索页**：
  - [ ] 搜索输入框与历史搜索记录记录。
  - [ ] 联想与模糊查询逻辑（与后端外号词库映射对接，支持搜“42”、“翼德”）。
- [ ] **搜索结果流**：复用首页的干员卡片组件以呈现搜索结果。

## 4. 干员详情能力展板 (Operator Detail)

- [ ] **干员头部信息区**：高清半身/全身立绘，干员名称、职业、总杯级评定标识。
- [ ] **雷达图与能力值看板**：
  - [ ] 列出该干员在各项子榜单（1000甲挂机、决战总伤、等）的具体名次与数值。
- [ ] **下钻明细与透明公式弹窗**：
  - [ ] 支持点击各项数据项，弹出具体运算公式或环境基准提示（例如：`1000甲 * 2.6最终倍率`），展示数据的由来。

## 5. 交互与联调

- [ ] 对接后端提供的各个 HTTP API 接口。
- [ ] 数据流与页面响应的真机测试（处理大数据量渲染卡顿与瀑布流懒加载优化）。
- [ ] 对接云开发的静态资源存储（云存干员头像、立绘等图片）。

> 备注：该任务单为初期基于原型的概要设计，后续根据完整的视觉与补充设计需求进行细化调整。
