# 杯级计算器：后端重构与业务逻辑实施汇总报告 (2026-06-29)

## 一、 项目环境与架构体系重组

- **目录规划与重组**：建立 `data/` 下 `Operators`, `Enemies`, `raw_data`, `parsed_data` 的标准化分层结构。
- **模块化包管理**：将 `backend` 划分为 `src/crawler` (爬虫)、`src/parser` (解析)、`function/logic` (干员逻辑抽象)、`function/battle` (宏观战斗公式) 以及 `main` (入口与API服务)。
- **微服务化 API**：引入了 FastAPI 并搭建在 `backend/main` 下，负责暴露核心 `/api/simulate` 等战斗仿真服务端口。

## 二、 自动化数据流水线落地

- **全量抓取**：编写并执行了明日方舟六星干员官方解包名单与 PRTS Wiki 的抓取脚本，原始文本统一至 `data/raw_data`。
- **大模型精准清洗**：利用 LLM 提示词提取包含特性、基础面板、信赖加成、精二满级天赋、及含有初始与消耗技力的技能组。
- **容错与格式对齐**：修复了抓取过程中 NoneType 报错的问题，清洗无效数据。最终获得 130 份高质量标准化 JSON，并将干员 ID 与解包编号强关联（例如 `AA00`）。

## 三、 面向对象干员逻辑动态生成与抽象

- **动态基类构建 (`builder.py`)**：读取 `parsed_data`，在 `function/logic/professions.py` 动态生成或复用各职业父类（如“重剑手”、“剑豪”、“神射手”等）。
- **对象自动生成**：自动在 `function/logic/operators/` 下吐出 130 位六星干员的独立 Python 子类文件骨架，完全解耦。

## 四、 任务推进成果

- 成功打通从“网页抓取” -> “大模型语义提取” -> “本地结构化存储” -> “根据数据反向智能生成 Python 对象树” 的全数据链路。
- 该底座完全适应后期的所有战斗逻辑补充（见《后端核心重构与战斗协同系统交付报告》）。
