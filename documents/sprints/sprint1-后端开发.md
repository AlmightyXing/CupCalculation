# Sprint 1: 后端开发阶段

## 1. 目标与范围
本阶段的核心目标是构建《明日方舟》干员数值计算的核心引擎和基础后端架构。涵盖从干员原始数据的解析、伤害和治疗公式的面向对象建模，到自动化大模型流水线的搭建。

## 2. 关键实现与里程碑
- **数据结构与解析模型**：设计并实现了基于 Pydantic 的干员技能和属性模型（`models.py`）。
- **计算核心开发**：
  - 构建了四大属性修饰器（基础加算、基础乘算、最终加算、最终乘算）的复合运算系统。
  - 完成了五种伤害类型（物理、法术、真实、治疗、护盾）的公式实现（`damage.py`，`base_operator.py`）。
- **自动化流与大模型接入**：
  - 成功接入 DeepSeek/Gemini API，编写 Prompt 实现对 PRTS 原始维基数据的自动化抓取与结构化提取。
  - 利用 Python AST 动态生成所有干员的独立计算逻辑类。
- **后端 API 搭建**：
  - 使用 FastAPI 搭建轻量级服务端，提供了 `/api/operators` 和核心演算接口的雏形。

## 3. 遇到的问题与解决方案
- **数据格式异构问题**：干员特殊技能机制繁杂，LLM 解析可能存在字段遗漏。解决方案：设计了严格的 Pydantic 校验和默认值后备策略，并在生成的代码中加入了安全机制。
- **动态类加载**：需要运行时尚未确定的干员脚本。解决方案：开发了 `operator_repo.py` 中的 `importlib` 动态加载机制，支持自动反射加载所有 `Operator` 子类。

## 4. 下阶段计划
- 核心后端演算逻辑经测试验证无误，将转向前端可视化界面（Vanilla JS + CSS）的开发。
