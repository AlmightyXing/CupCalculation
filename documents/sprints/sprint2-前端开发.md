# Sprint 2: 前端开发阶段

## 1. 目标与范围
本阶段旨在为《杯级计算器》打造一个现代、高级且交互流畅的可视化用户界面。采用 Vanilla HTML/CSS/JS 开发，确保应用结构轻量化，并在后续打包为独立桌面应用。

## 2. 关键实现与里程碑
- **现代美学与 UI 架构搭建**：
  - 设计了基于暗色模式 (Dark Mode) 的整体风格，引入毛玻璃 (Glassmorphism) 和动态微交互特效，确保达到要求的高级感。
  - 完成了左侧全局侧边栏导航和右侧内容展示区（全境阵型模拟、单干员沙盒、总览排名）。
- **干员筛选与展示**：
  - 实现了干员大职业与小分支的级联选择下拉框，并接入动态模糊搜索。
  - 干员卡片（`operator-card`）展示头像、职业和稀有度，支持点击选中并高亮。
- **沙盒与排名视图**：
  - 沙盒模拟界面（Sandbox）支持选择特定敌人属性预设，提供一键伤害演算与 DPS 可视化报告。
  - 杯级排名与输出视图提供了基于计算引擎结果的横向 DPS 对比。

## 3. 遇到的问题与解决方案
- **UI 动态交互与布局溢出**：大量干员卡片展示在部分分辨率下出现布局错乱。解决方案：使用 CSS Grid 和 Flexbox 组合布局，优化了滚动条样式与动态卡片悬浮过渡效果。
- **资源静态引用问题**：原生 JS 无法像 Webpack 一样打包资源。解决方案：设定了严格的文件目录树和绝对相对路径标准，为后续接入 `pywebview` 奠定基础。

## 4. 下阶段计划
- 前端基础页面完成，接下来将通过 Fetch API 与 FastAPI 后端进行全方位对接，实现数据的实时互通与沙盒实装验证。
