# Sprint 3: 前后端联通及数据互通阶段

## 1. 目标与范围
本阶段将 Sprint 1 产出的 FastAPI 计算后端与 Sprint 2 设计的 HTML/JS 界面深度融合。实现跨域通信、静态资源加载，打通干员数据的实时存取以及沙盒和排名运算的动态结果呈现。

## 2. 关键实现与里程碑
- **桌面端封装**：
  - 引入了 `pywebview` 框架，在后端启动时同步建立本地桌面窗口，无缝嵌套前端 HTML 页面。
  - 配置了 FastAPI 的静态文件挂载点（`app.mount`），使得浏览器容器能够直接读取本地的 CSS/JS 文件与头像图片（`data/avatars_head_portrait`）。
- **核心接口互通**：
  - 实现了 `/api/operators` 接口，前端 `app.js` 能够异步获取内存中热加载的 120+ 位干员列表与技能信息，并注入级联筛选框。
  - 完成了 `/api/sandbox/simulate` 与 `/api/simulate` 接口的 POST 请求与返回。前端可实时提交表单参数并渲染后端的 DPS 伤害结果。
- **系统连调与排错**：
  - 成功跑通“选择干员 -> 调整预设 -> 提交运算 -> 展示伤害面板”的完整链路。

## 3. 遇到的问题与解决方案
- **异步端口冲突与页面加载时序**：后端在特定端口启动需要时间，WebView 页面可能在接口就绪前完成加载。解决方案：编写了 `get_free_port()` 动态获取空闲端口，并通过线程 `daemon` 阻塞配合，确保后端启动完毕后再加载前端窗口。
- **大小写导致的干员丢失报错**：部分干员如 W (b214_w.py) 与 麒麟R夜刀 (mh02_麒麟r夜刀.py) 因代码命名和 JSON 大小写不匹配而报错。解决方案：重构 `operator_repo.py` 将干员映射关系强制转换为全小写。

## 4. 下阶段计划
- 全局业务链路已打通，下阶段将聚焦系统的易用性打磨、错误边界处理、以及提供完全独立的打包封装方案。
