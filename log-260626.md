# Phase 2 爬虫与数据清洗阶段开发日志

**时间戳：2026-06-26 15:11**

## 一、新增依赖项
在虚拟环境中安装了以下依赖：
- `beautifulsoup4`, `requests`, `lxml`：用于抓取并解析 PRTS 网页 DOM，剥离无关的 HTML 标签。
- `python-dotenv`：用于在本地安全地配置和读取环境变量。

## 二、新增及修改文件清单

### 1. `src/crawler/prts_crawler.py`
- **功能**：网页抓取与预处理工具。
- **内容**：提供 `fetch_operator_page()` 爬取网页 HTML；`extract_relevant_text()` 用于寻找主容器内容（`mw-parser-output`），剔除无用标签并提取干净纯文本，以便喂给 LLM。

### 2. `src/crawler/llm_cleaner.py`
- **功能**：大模型结构化清洗模块。
- **内容**：封装了对 DeepSeek API 的调用逻辑，设定了精确的 `SYSTEM_PROMPT`。该系统提示词指定了从纯文本中提取干员姓名、职业、精二满级不含信赖的面版数据、信赖加成、天赋和技能信息，并转换为 `character_sample.json` 结构的有效 JSON 格式。
- **更新**：新增了对 `dotenv` 的支持，自动从根目录加载 `.env` 获取 `DEEPSEEK_API_KEY`。

### 3. `src/crawler/fetch_character.py`
- **功能**：整合流程的入口测试脚本。
- **内容**：可从终端接收干员名称参数，依次调用抓取、LLM 清洗逻辑，最终将 JSON 保存至 `data/processed/` 目录。

### 4. 环境变量配置
- 新建了根目录下的 `.env` 文件作为模板，用户可填入真实的 `DEEPSEEK_API_KEY`。
- 修改了 `.gitignore`，加入了 `.env`, `.venv/` 以及 `__pycache__/`，确保敏感信息和临时环境文件不会被推送到代码仓库。

## 四、优化大模型参数捕获方案 (Phase 1.5)

**时间戳：2026-06-26 15:45**

- **问题背景**：之前单一的 Prompt 让大模型同时处理角色特性、多个天赋和技能时，容易出现信息遗漏（例如未能捕获解放者的 +200% 攻击力，以及各类天赋加成）。
- **解决方案**：重构了 `llm_parser.py`，实施了分而治之的策略。
- **实施细节**：
  1. 增加了 `TRAIT_PROMPT_TEMPLATE` 和 `TALENT_PROMPT_TEMPLATE`，在请求阶段让大模型针对特性、各个天赋以及单独的技能，分别生成对应且纯净的 `SkillParams` 修正 JSON。
  2. 在 `parse_character_skills` 流程中，按粒度独立生成并缓存特性、天赋和技能的文件（如 `char_xxx_trait.json`, `char_xxx_talent0.json`）。
  3. 新增了 `_merge_params` 合并算法：能够将不同维度的修正器按照游戏底层数值逻辑（加算叠加，倍率相乘）拼装在一起，并最终喂给 `attribute.py` 中的计算引擎。
- **结果验证**：对能天使、银灰、玛恩纳执行了全量重算。三大类复杂干员的常驻增益（天赋/特性）和条件触发增益均已被正确抽离和叠加合并。
